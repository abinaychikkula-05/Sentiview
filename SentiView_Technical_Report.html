Project Overview

SentiView is a full-stack web application that helps businesses analyze client feedback and understand customer sentiment in real-time.

What is SentiView?

SentiView allows users to:

  • Upload customer feedback via CSV files
  • Add feedback manually through a simple form
  • Get automatic sentiment analysis (Positive, Negative, Neutral)
  • View interactive charts and analytics
  • Manage user accounts and permissions
  • Track feedback over time with beautiful visualizations

Database Structure

User Schema

{ username: String (unique), email: String (unique), password: String (hashed), company: String, role: String (enum: 'user', 'admin'), createdAt: Date, updatedAt: Date }

Feedback Schema

{ userId: ObjectId, clientName: String, feedback: String, sentiment: { label: String ('Positive', 'Negative', 'Neutral'), score: Number (-1 to +1), confidence: Number (0 to 1) }, rating: Number, category: String, createdAt: Date, updatedAt: Date }

Authentication & Authorization

Authentication Flow

Step Action Response
1 User submits login credentials Frontend sends POST /api/auth/login
2 Backend verifies email & password Compares with hashed password in DB
3 JWT token generated Token includes userId and role
4 Token stored in frontend Sent in Authorization header for requests
5 Protected routes verify token Middleware checks validity and role

JWT Token Structure

Header: { "alg": "HS256", "typ": "JWT" } Payload: { "userId": "6929c094ce57749d4bf3c0c3", "role": "admin", "iat": 1764350473, "exp": 1764955273 }

Challenges & Solutions

Challenge 1: MongoDB Connection

Problem: MongoDB Atlas connection failed with DNS errors in dev environment
Solution: Implemented mongodb-memory-server for local development, uses in-memory MongoDB

Challenge 2: Dashboard Not Loading

Problem: Feedback data wasn't displaying after login
Solution: Fixed Dashboard component to check isAuthenticated && token from context instead of user object

Challenge 3: Dark Mode Contrast

Problem: Text was invisible in dark mode (white on white)
Solution: Added comprehensive dark mode CSS with proper background colors (#3a3a4d) and text contrast

Challenge 4: Admin Role Not Working

Problem: Admin couldn't access admin panel despite being admin
Solution: Three fixes:
  • Updated User model schema to include role field
  • Modified JWT generation to include role in token payload
  • Fixed database record (admin user had role='user')
